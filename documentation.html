User API Documentation

This API allows users to register, deregister and manage their user data.

  • Register

    Endpoint URL: "/users"

    Authorization: none

    Method: POST

    Params: none

    Body: JSON object

    Example body: { username: "", password: "", email: "", registrationCode: "" }

    Response: JSON object Status Success: 201

  • Login

    Endpoint URL: "/users/login"

    Authorization: Basic HTTP

    Method: POST

    Params: none

    Body: JSON

    Example body: {"username": "", "password":"" }

    Response: JSON (comtains JWT) Status Success: 200

  • User Information

    Endpoint URL: "/users/:username"

    Authorization: Bearer JWT

    Method: GET

    Params: :username (required)

    Body: none

    Example body: none

    Response: JSON Status Success: 200

  • Update user

    Endpoint URL: "/users/:username"

    Authorization: Bearer JWT

    Method: PUT

    Params: :username (required)

    Body: JSON object

    Example body: new User(loginCredentials, contactInformation, billingInformation, userGroups)

    Response: JSON Status Success: 201

  • Delete user

    Endpoint URL: "/users/:username"

    Authorization: Bearer JWT

    Method: DELETE

    Params: :username (required)

    Body: none

    Example body: none

    Response: Text Status Success: 201

  • User Schema

    new User(loginCredentials, contactInformation, billingInformation, userGroups)


    {
    "loginCredentials": { "username" : "", "password" : "", "email" : "" }

    "contactInformation": { "name" : "", "surname" : "",
    "address" : { "street" : "", "number" : "", "zip" : "", "city" : "", "county" : "", "country" : "" }
    }

    "billingInformation": { "bank" : "", "iban" : "" }

    "userGroups": string[ ]
    }

  • Config

    Database password and jwt secret are collected as environment variables

    Database password env: databasePw

    JWT Secret env: user_api_secret
